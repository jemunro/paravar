# DESIGN.md

Implementation notes for contributors to blocky. Covers the sharding algorithm, threading model, BGZF I/O, index parsing, and the layout of the source tree. For installation and CLI usage see [README.md](README.md).

## 1. Overview

blocky parallelises processing of BGZF-compressed files by sharding them along record boundaries. A BGZF file is a sequence of independent gzip members, each containing up to 65 536 bytes of uncompressed data, terminated by a 28-byte empty EOF block. Each block is self-contained at the *compression* level, but **records routinely span multiple blocks**, so a shard boundary cannot fall at an arbitrary block boundary — it must land on a record boundary inside one of those blocks.

To find that record boundary, blocky either:
- decompresses one BGZF block near the desired split point and `memchr`'s for `\n` (text/VCF, scan-mode), or
- reads record start offsets directly from a CSI/TBI/GZI index (index-mode — required for BCF, whose records have no in-band delimiter).

The chosen boundary block is then decompressed, split at the record boundary, and the two halves are recompressed (the tail closes the previous shard; the head opens the next). For an N-shard split, exactly N-1 boundary blocks are decompressed and recompressed. All blocks *between* boundaries are byte-copied at disk bandwidth without ever entering libdeflate.

## 3. Module responsibilities

### `main.nim` — CLI dispatch

- Top-level `mainEntry()` dispatches to `runScatter`, `runRun`, `runGather`, `runCompress`, `runDecompress`.
- Uses `std/parseopt` with a `ShortNoVal` set so attached values (`-n4`) work alongside separated (`-n 4`).
- Argument validation, file existence checks, and helpful error-to-stderr live here. Subcommand-specific flag tables are described in README.
- For `run`, argv is split at `:::` / `---` separators into `blockyArgs` + a list of pipeline stages; each stage becomes one process in an `sh -c "a | b | c"` chain.
- `{}` placeholder substitution and `\{}` escape live here (see `substituteToken`).

### `bgzf.nim` — BGZF format + indexes

The largest and most foundational module. No external deps except libdeflate (vendored).

Key responsibilities:

- **Codec layer.** Thin FFI over libdeflate (`libdeflate_alloc_compressor`, `libdeflate_deflate_compress`, `libdeflate_crc32`, decompressor counterparts). Codecs are expensive to allocate, so each thread caches one compressor and one decompressor in `{.threadvar.}` slots. The compressor's level is tracked and reset on change.
- **Block scanning.** `scanBgzfBlockStarts(path, startAt, endAt)` walks the file sequentially and returns offsets of every block start. `bgzfBlockSize(buf)` parses the 18-byte header and returns the total compressed block size (or -1 if invalid).
- **Block I/O.** `decompressBgzf` / `decompressBgzfInto`, `compressToBgzf` / `compressToBgzfMulti`, `splitBgzfBlockBothSides` (decompress a block, split its uncompressed payload at a given offset, return both halves as new compressed blocks).
- **Zero-copy transfer.** `copyRange[FromFile]` uses a tiered fallback: `copy_file_range` (Linux ≥ 4.5, CoW on many FSes) → `sendfile` (works across more kernels) → `pread`/`pwrite` loop. Permanent-failure flags are cached as relaxed atomics so a single fallback decision propagates across threads.
- **Format sniffing.** `inferInputFormat(path)` first inspects the extension, then falls back to `sniffFormat(bytes)` on the first decompressed block (BCF magic `BCF\x02\x02`, or `##fileformat` for VCF, else text).
- **Header extraction.** `getHeaderAndFirstBlock` (VCF) and `extractBcfHeaderAndFirstOffset` (BCF) return the header bytes and the file offset of the first non-header block.
- **Index parsing.** Three formats:
  - `parseTbiVirtualOffsets(tbiPath)` — TBI (`TBI\x01` magic): per-reference bins → chunks of `(start_vo, end_vo)`. Returns sorted unique `(file_offset, within_block_offset)` pairs.
  - `parseCsiVirtualOffsets(csiPath)` — CSI (`CSI\x01` magic), hierarchical binning controlled by `min_shift`/`depth`. Required for BCF.
  - `parseGziBlockStarts(gziPath)` — bgzip GZI: `uint64` count followed by `(compressed_offset, uncompressed_offset)` pairs. No within-block offsets.
  - Helper `readIndexVirtualOffsets` tries CSI first, then TBI, then GZI.
- **Virtual offset encoding:** `(file_offset << 16) | within_block_offset`.

### `scatter.nim` — sharding algorithm

- `ShardTask` is the on-disk plan for one shard: header bytes, an optional left boundary block (decompressed + recompressed tail of the previous boundary block), a `(file_offset, length)` range of middle blocks to byte-copy, an optional right boundary block, and the BGZF EOF block.
- `computeShards(path, nShards, nThreads, forceScan, fmt, clamp)` produces the list of `ShardTask`. Two strategies:
  - **Index-mode** (default when an index is present and `--scan` is not set): compute N evenly spaced target file offsets within `[dataStart, dataEnd]`, binary-search each into the sorted virtual offsets, then dispatch boundary blocks in parallel to a pool of boundary workers (atomic pull counter `gBoundaryNext`). Each worker decompresses one block, splits at its within-block offset, and emits two recompressed halves.
  - **Scan-mode** (`--scan`, or no index for VCF/text — BCF without CSI errors out): scan all block starts, divide them evenly, and for each candidate boundary decompress the block and `memchr` for `\n`. If no newline in that block, expand bidirectionally to neighboring blocks.
  - `--clamp`: if fewer split points than requested shards exist, reduce N (with a stderr warning) instead of erroring.
- `scatter(...)` is the high-level entry for the `scatter` subcommand: calls `computeShards`, then spawns scatter workers that each read their `ShardTask` and write a single shard output file using `copyRange` for the middle range.
- Output template resolution: `shardOutputPath(tmpl, idx, total)` substitutes `{}` with a zero-padded shard number; parent directories are created on demand.

### `run.nim` — worker pool + concat

The orchestration for `blocky run`. Per-shard pipeline:

1. **Scatter phase.** `computeShards` produces `nWorkers * maxShardsPerWorker` shard tasks.
2. **Worker threads.** `nWorkers` OS threads, each looping until exhausted:
   - Pull the next shard index with `gNextShard.fetchAdd(1, moRelaxed)`.
   - Fork `sh -c "stage1 | stage2 | …"` via `posix_spawn`/`fork`+`execvp`. Stages were built per-shard via `buildShellCmdForShard` so `{}` resolves to this shard's number.
   - Stream the decompressed shard contents to the subprocess's stdin pipe (`ShardTask.decompress = true`).
   - An **interceptor thread** reads the subprocess's stdout, sniffs whether the output is BGZF or raw, strips duplicate headers for shards 2..N, recompresses to BGZF if needed, and writes a tmp file.
   - On completion the worker deposits the tmp path into the shared `DepositQueue` slot for its shard index and atomically sets that slot's `ready` flag.
3. **Concat thread.** Walks slots 0..total-1 in order, spin-waits (1 ms sleep) on `ready`, copies the tmp file's content into the final output (stripping the EOF block from all but the last), then deletes the tmp file.

Run modes (`RunMode`):

- `rmFile` — `-o path` set; concat thread writes the final assembled output.
- `rmStdout` — no `-o`, no `--discard`; concat thread writes to stdout.
- `rmDiscard` — `--discard` set; the tool manages its own output via `{}` placeholders; no concat is performed.
- `rmFile + --discard` is rejected by `main.nim`.

Shared state (all atomics):

- `gNextShard: Atomic[int]` — work distribution.
- `gAnyFailed: Atomic[bool]` — early-abort signal (any worker on non-zero exit). Workers check it before pulling the next shard unless `--no-kill`.
- `gShardsWithOutput: Atomic[int]` — used to warn if `{}` was specified but the tool produced no output.

Linux-specific niceties: `F_SETPIPE_SZ` enlarges subprocess pipes (~256 KiB) to reduce blocking; all pipes have `FD_CLOEXEC` to keep them out of grandchild processes.

Optimisation: if `RunMode == rmFile` and shard 0 is being processed, the worker may write directly to the final output FD and skip the tmp-file round-trip.

### `gather.nim` — pre-existing shard concat

- `gatherFiles(cfg, inputPaths)` walks each input, strips its header (and `#CHROM` line) on shards 2..N, byte-copies its block range, and writes a single BGZF EOF at the end.
- Before any output is written, `extractChromLine` is called on every input and the `#CHROM` lines are compared byte-for-byte. A mismatch exits 1 with no partial output.
- `inferFileFormat(path, override)` chooses VCF vs BCF vs text from extension first, falling back to magic-byte sniff via `bgzf.sniffFormat`.

## 4. Threading model (run subcommand)

```
   main thread
        │
        ├── spawns N worker threads ──┐
        │                             │ each worker:
        │                             │   - pull shard idx (atomic)
        │                             │   - fork sh -c "stages"
        │                             │   - feed stdin from ShardTask
        │                             │   - spawn interceptor thread
        │                             │       (reads stdout → tmp file)
        │                             │   - deposit tmp path to DepositQueue
        │
        └── spawns 1 concat thread ───┘
                                      consumes DepositQueue in FIFO order,
                                      writes to -o / stdout, deletes tmp files
```

No mutexes — synchronisation is entirely through atomics and the lock-free `DepositQueue` (an `allocShared0` array of `DepositSlot { ready: Atomic[bool]; path: cstring }`). Polling uses a short sleep rather than condition variables for simplicity.

## 5. Sharding algorithm details

- **Targets:** with a data range `[dataStart, dataEnd]`, the i-th target offset is `dataStart + (i * (dataEnd - dataStart)) / N`.
- **Index-mode resolution:** binary-search each target in the sorted virtual-offsets array; the closest virtual offset that does not fall inside the header is used.
- **Scan-mode resolution:** if the boundary block does not contain a newline, expand to neighbors. For text/VCF, splits must land on a `\n`. For BCF this path is disabled — CSI is required.
- **Clamp semantics:** if the number of distinct valid split points is `< N`, error out unless `--clamp` is set (in which case N is reduced to the available split count).
- **Boundary parallelism:** boundary decompression is itself parallel — up to `nThreads` workers pull from `gBoundaryNext` and each handles one boundary block.

## 6. Index format quick reference

| Format | Magic       | Provides                                  | Parser                          |
|--------|-------------|-------------------------------------------|---------------------------------|
| TBI    | `TBI\x01`   | virtual offsets per bin/ref               | `parseTbiVirtualOffsets`        |
| CSI    | `CSI\x01`   | virtual offsets, hierarchical bins        | `parseCsiVirtualOffsets`        |
| GZI    | (binary)    | block-start offsets only                  | `parseGziBlockStarts`           |

Virtual offset bit layout: `(file_offset:int48 << 16) | within_block_offset:int16`.

## 7. BGZF invariants

- One block = 18-byte gzip header (with BGZF magic in `XLEN`/`SI1`/`SI2`) + DEFLATE-compressed payload + 4-byte CRC32 + 4-byte ISIZE.
- Uncompressed payload ≤ 65 536 bytes (in practice usually 64 KiB minus a small margin).
- The empty BGZF EOF block is 28 bytes, fixed bit pattern, always at end of a well-formed file.
- Block boundaries are valid *BGZF* boundaries but **not** valid record boundaries — records (VCF/BCF rows, BED lines, etc.) can span multiple blocks. Every shard split must land on a record boundary, which requires either decompressing the boundary block and locating it (newline for text formats) or reading it from an index (CSI for BCF).

## 8. Non-obvious patterns and invariants

- **Thread-local codec cache.** `libdeflate_alloc_compressor` allocates non-trivial memory; per-thread caching is mandatory for performance.
- **Only boundary blocks are touched.** For an N-shard split, exactly N-1 blocks are decompressed (to locate the record boundary inside them) and recompressed (split into a tail + head). Everything else is byte-copied.
- **Zero-copy tier fallback** in `copyRange` caches permanent-failure flags as relaxed atomics so a single decision propagates across threads.
- **Shard 0 fast path.** In `rmFile` mode, shard 0's worker may write directly to the final output FD (skipping its tmp file).
- **Subprocess stdin = decompressed.** Workers always send raw uncompressed bytes to the tool; the tool decides its own output format.
- **Subprocess stdout format is auto-detected.** The interceptor sniffs BGZF magic vs raw text and adapts (strips headers from shards 2..N either way; recompresses raw output to BGZF before writing the tmp file).
- **`{}` substitution rules.** `{}` becomes the zero-padded shard number; `\{}` is the literal escape. Validation is centralised in `substituteToken`. Mode matrix (`-o` × `--discard`) is enforced in `main.nim`.
- **`fetchAdd` for work distribution.** All "next item" counters use `moRelaxed`. Termination flags use `moAcquire`/`moRelease`.
- **lseek after pwrite.** `copyRange` explicitly advances the output FD position after `pwrite`, because callers use `tell` and sequential `write` afterwards and would otherwise see a stale position.

## 9. Build configuration

- [blocky.nimble](blocky.nimble):
  - `before build` hook runs `buildLibdeflate` (clones submodule if missing, configures with CMake, builds `vendor/libdeflate/build/libdeflate.a`) and `bgzip`s the LICENSE files into the binary's resource directory.
  - `task release` adds `--define:release --define:strip --panics:on`.
  - `task test` builds the binary (CLI tests shell out to it), clears `nimcache/tests`, then runs each `tests/test_*.nim` in turn.
- [config.nims](config.nims) auto-loads `nimble.paths` and disables panics for test executables so `AssertionDefect` is raised normally (otherwise `--panics:on` would convert assertion failures into raw process abort).
- libdeflate is linked statically via `{.passC: "-I vendor/libdeflate".}` and `{.passL: "vendor/libdeflate/build/libdeflate.a".}` inside `bgzf.nim`.

## 10. Testing

- [tests/test_utils.nim](tests/test_utils.nim) defines a `timed()` macro that catches `AssertionDefect`, prints PASS/FAIL with a label and timing, and accumulates failure counts.
- Tests are labelled (e.g. `B01`..`B06` for `test_bgzf`, `S01`..`S04` for `test_scatter`) so a failure in `nimble test` output points back to a specific block.
- Fixtures live under `tests/data/` and are regenerated by [tests/generate_fixtures.sh](tests/generate_fixtures.sh) (requires `bcftools`, `bgzip`, `tabix`).
- Running a single test:
  ```bash
  nim c --hints:off -r tests/test_scatter.nim
  ```
  If imported-module changes don't take effect, `rm -rf nimcache/tests` and retry.
