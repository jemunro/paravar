## What this is

**blocky** is a fast parallel processor for BGZF-compressed bioinformatics files (VCF, BCF, BED, GFF, etc.), written in Nim. It splits a bgzipped file into N shards and pipes each shard concurrently through a user-supplied tool pipeline (`bcftools view`, etc.), then concatenates the results in order. Records can span multiple BGZF blocks, so each shard boundary must land on a record boundary — found either by decompressing the boundary block and scanning for a newline (text/VCF), or by reading record offsets from a CSI index (required for BCF, since BCF records have no in-band delimiter). All blocks *between* boundaries are byte-copied without decompression, so only the N-1 boundary blocks per N-shard split are ever decompressed and recompressed.

## Repository layout

- [src/blocky.nim](src/blocky.nim) — binary entry point (just includes `blocky/main.nim`).
- [src/blocky/main.nim](src/blocky/main.nim) — CLI parsing and subcommand dispatch (`scatter`, `run`, `gather`, `compress`, `decompress`).
- [src/blocky/bgzf.nim](src/blocky/bgzf.nim) — BGZF I/O, libdeflate FFI, TBI/CSI/GZI index parsing, zero-copy file transfer.
- [src/blocky/scatter.nim](src/blocky/scatter.nim) — shard-boundary computation and parallel shard writing.
- [src/blocky/run.nim](src/blocky/run.nim) — worker pool, subprocess pipelines, in-order concat.
- [src/blocky/gather.nim](src/blocky/gather.nim) — concatenation of pre-existing shard files.
- [tests/test_*.nim](tests/) — test suite. Fixtures generated by [tests/generate_fixtures.sh](tests/generate_fixtures.sh).
- [vendor/libdeflate/](vendor/) — git submodule, built statically via CMake during nimble build.
- [blocky.nimble](blocky.nimble), [config.nims](config.nims) — build configuration.

## Build and test

```bash
nimble build              # debug binary
nimble release            # optimized + stripped (`--define:release --define:strip --panics:on`)
nimble test               # runs every tests/test_*.nim sequentially
nim c --hints:off -r tests/test_scatter.nim   # single test file
```

`nimble test` calls `rm -rf nimcache/tests` first — if you hand-run a single test and edits in imported modules aren't picked up, clear that cache.

Test runtime requirements (must be on `PATH`): `bcftools`, `bgzip`, `tabix`. Build requirements: Nim ≥ 2.0, cmake, a C compiler.

## Conventions

- Nim ≥ 2.0. No new external dependencies without prior discussion (see Contributing in README).
- BAM/CRAM are intentionally unsupported — reads span block boundaries, so block-level splitting would corrupt records.
- BCF always requires a CSI index; there is no scan fallback for BCF.
- Output `{}` placeholder is the zero-padded shard number; literal `{}` is escaped as `\{}`.
